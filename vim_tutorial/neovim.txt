gd: go to definition

If gd does not jump correctly to the definition, another way is to use ctags:
https://kulkarniamit.github.io/whatwhyhow/howto/use-vim-ctags.html

sudo apt-get update && sudo apt-get install -y exuberant-ctags
move to the project directory
ctags --recurse=yes --exclude=.git --exclude=BUILD --exclude=.svn --exclude=vendor/* --exclude=node_modules/* --exclude=db/* --exclude=log/* --exclude=build

vim -t tags [tags is the tags file generated from the command above] 

ctrl + ]: move to the tag (or the definition)

:bd close a file in buffer
