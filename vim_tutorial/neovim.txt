Tested on Debian 12:

(Source:
# install neovim: 
https://github.com/neovim/neovim/releases
# install clangd:
https://github.com/clangd/clangd/releases
# configure neovim via kickstart.nvim
https://github.com/nvim-lua/kickstart.nvim
)

sudo apt update

sudo apt install cmake clangd-19 wget git build-essential libclang-dev

sudo apt install bison byacc

sudo apt install make gcc ripgrep fd-find unzip git xclip curl


curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh  (to have the command cargo below - package manager for rust)
cargo install --locked tree-sitter-cli 

# Now we install nvim
curl -LO https://github.com/neovim/neovim/releases/latest/download/nvim-linux-x86_64.tar.gz
sudo rm -rf /opt/nvim-linux-x86_64
sudo mkdir -p /opt/nvim-linux-x86_64
sudo chmod a+rX /opt/nvim-linux-x86_64
sudo tar -C /opt -xzf nvim-linux-x86_64.tar.gz

# make it available in /usr/local/bin, distro installs to /usr/bin
sudo ln -sf /opt/nvim-linux-x86_64/bin/nvim /usr/local/bin/


# Clone kickstart.nvim
git clone https://github.com/nvim-lua/kickstart.nvim.git "${XDG_CONFIG_HOME:-$HOME/.config}"/nvim

#start Neovim
nvim


#enable clangd in nvim config: ~/.config/nvim/init.lua
vim.lsp.enable("clangd") or similar in this config file


An example to generate compile_commands.json for linux kernel using bear:
sudo apt install bear
(install some packages before to build linux-6.5.y:) 
sudo apt install libssl-dev bc libelf-dev flex
git clone --depth 1 --branch linux-6.5.y git://git.kernel.org/pub/scm/linux/kernel/git/stable/linux.git linux-kernel-6.5.y
cd linux-kernel-6.5.y
make mrproper
make defconfig
bear -- make -j4



view $MYVIMRC for more detail while opening nvim

<space><space>: show a list of files in buffer (currently being edited)
<space><s><f>: search files

gd: go to definition or simply right click on the word and choose "go to definition" (but the files compile_commands.json needs to be generated beforehand)

If gd does not jump correctly to the definition, another way is to use ctags:
https://kulkarniamit.github.io/whatwhyhow/howto/use-vim-ctags.html

sudo apt-get update && sudo apt-get install -y exuberant-ctags
move to the project directory
ctags --recurse=yes --exclude=.git --exclude=BUILD --exclude=.svn --exclude=vendor/* --exclude=node_modules/* --exclude=db/* --exclude=log/* --exclude=build

vim -t tags [tags is the tags file generated from the command above] 

ctrl + ]: move to the tag (or the definition)

:bd close a file in buffer
